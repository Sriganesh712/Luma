  1. PROJECT OVERVIEW
================================================================================

  Luma is a full-stack Learning Management System (LMS) built for
  educational institutions. It supports three user roles — Admin, Teacher, and
  Student — each with a dedicated portal. The platform's centrepiece is an
  AI mentor chat powered by GPT-4o (via GitHub Models) that gives students
  personalised academic help and emotional/wellbeing support.

  Core capabilities:
  ─ Institution management (multi-tenant via institution codes)
  ─ Class creation, student enrolment, teacher assignment
  ─ Study material uploads (PDF, DOCX, PPTX, video, links)
  ─ AI-generated MCQ / written assignments from any topic or uploaded material
  ─ Student assignment submission with automatic MCQ grading
  ─ AI-powered evaluation of written answers with per-question feedback
  ─ Persistent AI mentor chat (Study Mode + Wellbeing/Support Mode)
  ─ PDF upload to chat for document-aware Q&A
  ─ Grade tracking with per-class performance summaries
  ─ Real-time streaming AI responses (Server-Sent Events)
  ─ Role-based access control enforced at both API and database level

  └──────────────────────────┬─────────────────────────────────┘
                             │  REST + SSE
  ┌──────────────────────────▼─────────────────────────────────┐
  │               Express.js Backend  (Node.js ESM)            │
  │   Auth · Admin · Materials · Assignments · AI · Chat       │
  └─────────────┬────────────────────────────┬─────────────────┘
                │  Supabase JS SDK           │  OpenAI SDK
  ┌─────────────▼───────────────┐  ┌─────────▼──────────────────┐
  │  Supabase (PostgreSQL)      │  │  GitHub Models / GPT-4o    │
  │  Auth · RLS · Storage       │  │  (openai/gpt-4o endpoint)  │
  └─────────────────────────────┘  └────────────────────────────┘

  Data flow:
  • All reads/writes to user data go through Supabase (direct from frontend
    for reads, via backend for writes that require privileged access or AI).
  • The backend acts as a secure middleware for: registration, AI chat, PDF
    processing, assignment generation, and grading.
  • Row Level Security (RLS) on Supabase ensures users can only access data
    belonging to their own institution.

  5. USER ROLES & PERMISSIONS
================================================================================

  ADMIN
  ─────────────────────────────────────────────────────────
  • Creates the institution (gets a unique 6-character institution code)
  • Manages all classes: create, update, delete, assign teachers
  • Enrolls / unenrolls students in classes
  • Views all users (teachers, students) within the institution
  • Institution code is used by teachers/students to join the platform
  • Full read access to all institution data

  TEACHER
  ─────────────────────────────────────────────────────────
  • Manages their assigned classes (view class, student roster)
  • Uploads study materials (PDF, PPTX, DOCX, video, external link)
  • Creates assignments (MCQ, written, or mixed)
  • Uses AI to auto-generate assignment questions from topic + materials
  • Publishes / closes assignments
  • Reviews student submissions
  • Grades written answers manually or triggers AI evaluation
  • Views submission statistics

  STUDENT
  ─────────────────────────────────────────────────────────
  • Views enrolled classes and their materials
  • Takes published assignments (MCQ auto-graded, written saved for grading)
  • Views own grades and per-class performance averages
  • Uses Luma chat (Study Mode + Wellbeing/Support Mode)
  • Uploads PDF documents to chat for document-aware Q&A
  • Chat history persisted across sessions (up to 30 sessions, 20 msg history)

  7. BACKEND — API ENDPOINTS

  Base URL: http://localhost:3000  (or VITE_API_URL in .env)

  HEALTH
  ─────────────────────────────────────────────────────────
  GET  /                           "Luma Backend Running 🚀"
  GET  /api/health                 { status: "ok", ts: "..." }

  AUTH   (no auth required)
  ─────────────────────────────────────────────────────────
  POST /api/auth/register          Register user + create/join institution
    Body: { email, password, name, role,
            institutionName? (admin) | institutionCode? (teacher/student) }
    – Creates Supabase auth user
    – Creates/looks up institution record
    – Inserts into public.users with institution_id

  CHAT  (no auth required — anonymous AI access)
  ─────────────────────────────────────────────────────────
  POST /api/chat                   Non-streaming chat (legacy)
    Body: { message, history[], pdfText?, mode? }

  POST /api/chat/stream            SSE streaming chat  ← primary endpoint
    Body: { message, history[], pdfText?, classId?, mode? }
    Response: text/event-stream  →  data: { content: "..." }
                                    data: { done: true }
    – Streams GPT-4o tokens in real time
    – Supports Study mode (academic) and Support mode (wellbeing)
    – If classId provided, fetches class materials as context

  POST /api/upload-pdf             Extract text from PDF
    Content-Type: multipart/form-data  field: "file"
    Response: { text: "..." }  (up to 100,000 chars)

  ADMIN  (requireAuth + requireRole("admin"))
  ─────────────────────────────────────────────────────────
  GET    /api/admin/stats                    Institution stats

  MATERIALS  (requireAuth)

  ASSIGNMENTS  (requireAuth)
  ─────────────────────────────────────────────────────────
  GET    /api/assignments                    List assignments
  GET    /api/assignments/:id                Get assignment + questions

  POST /api/ai/evaluate-submission/:id  AI-grade written answers  [teacher/admin]
    – Sends all written answers to GPT-4o for scoring + feedback
    – Updates answers.score and answers.ai_feedback in DB

  8. DATABASE SCHEMA
================================================================================

  All tables live in the "public" schema in Supabase (PostgreSQL).
  Row Level Security (RLS) is ENABLED on all tables.

  TABLE: institutions
  ─────────────────────────────────────────────────────────
  id           uuid  PK  (auto-generated)
  name         text  NOT NULL
  code         text  UNIQUE  (6-char institution join code, e.g. "A1B2C3")
  logo_url     text
  created_at   timestamptz

  TABLE: users
  ─────────────────────────────────────────────────────────
  id             uuid  PK  →  auth.users(id)
  institution_id uuid  →  institutions(id)
  name           text  NOT NULL
  email          text  NOT NULL
  role           text  CHECK IN ('admin', 'teacher', 'student')
  avatar_url     text
  created_at     timestamptz

  TABLE: classes
  ─────────────────────────────────────────────────────────
  id             uuid  PK
  institution_id uuid  →  institutions(id)  NOT NULL
  name           text  NOT NULL
  subject        text
  teacher_id     uuid  →  users(id)  (nullable: class can exist without teacher)
  created_at     timestamptz

  TABLE: enrollments  (junction: students ↔ classes)
  ─────────────────────────────────────────────────────────
  id          uuid  PK
  class_id    uuid  →  classes(id)  NOT NULL
  student_id  uuid  →  users(id)  NOT NULL
  enrolled_at timestamptz
  UNIQUE (class_id, student_id)

  TABLE: study_materials
  ─────────────────────────────────────────────────────────
  id               uuid  PK
  class_id         uuid  →  classes(id)  NOT NULL
  teacher_id       uuid  →  users(id)
  title            text  NOT NULL
  type             text  CHECK IN ('pdf','pptx','docx','video','link','other')
  file_url         text  (Supabase Storage URL)
  external_url     text
  file_size_bytes  bigint
  description      text
  created_at       timestamptz

  TABLE: assignments
  ─────────────────────────────────────────────────────────
  id           uuid  PK
  class_id     uuid  →  classes(id)  NOT NULL
  teacher_id   uuid  →  users(id)
  title        text  NOT NULL
  description  text
  type         text  CHECK IN ('mcq', 'written', 'mixed')
  total_points integer  DEFAULT 100
  deadline     timestamptz
  status       text  DEFAULT 'draft'  CHECK IN ('draft','published','closed')
  created_at   timestamptz

  TABLE: assignment_targets  (class-wide or individual targeting)
  ─────────────────────────────────────────────────────────
  id            uuid  PK
  assignment_id uuid  →  assignments(id)  NOT NULL
  target_type   text  CHECK IN ('class', 'student')
  target_id     uuid  (class_id or student_id)

  TABLE: questions
  ─────────────────────────────────────────────────────────
  id             uuid  PK
  assignment_id  uuid  →  assignments(id)  NOT NULL
  question_text  text  NOT NULL
  type           text  CHECK IN ('mcq', 'written')
  options        jsonb  (MCQ: [{ label: "A", text: "..." }, ...])
  correct_answer text   (MCQ: "A"/"B"/"C"/"D")
  points         integer  DEFAULT 10
  order_index    integer  DEFAULT 0

  TABLE: submissions
  ─────────────────────────────────────────────────────────
  id            uuid  PK
  assignment_id uuid  →  assignments(id)  NOT NULL
  student_id    uuid  →  users(id)  NOT NULL
  submitted_at  timestamptz
  total_score   integer  (null until graded)
  status        text  DEFAULT 'submitted'  CHECK IN ('submitted','graded')
  UNIQUE (assignment_id, student_id)

  TABLE: answers
  ─────────────────────────────────────────────────────────
  id               uuid  PK
  submission_id    uuid  →  submissions(id)  NOT NULL
  question_id      uuid  →  questions(id)  NOT NULL
  student_answer   text
  score            integer
  ai_feedback      text  (GPT-4o generated feedback per question)
  teacher_feedback text  (manual override)

  TABLE: chat_sessions
  ─────────────────────────────────────────────────────────
  id         uuid  PK
  student_id uuid  →  users(id)  NOT NULL
  class_id   uuid  →  classes(id)  (optional class context)
  title      text  (first 50 chars of first message)
  created_at timestamptz
  updated_at timestamptz

  TABLE: chat_messages
  ─────────────────────────────────────────────────────────
  id         uuid  PK
  session_id uuid  →  chat_sessions(id)  NOT NULL
  role       text  CHECK IN ('user', 'assistant')
  content    text  NOT NULL
  created_at timestamptz

  RLS POLICY SUMMARY
  ─────────────────────────────────────────────────────────
  • Users only see data from their own institution
  • Students only see their own submissions, answers, chat sessions
  • Teachers only manage classes/materials/assignments they own
  • Admins have full access within their institution
  • Helper functions: my_institution_id() and my_role() for policy evaluation

  INDEXES (performance)
  ─────────────────────────────────────────────────────────
  users(institution_id), classes(institution_id, teacher_id),
  enrollments(class_id, student_id), study_materials(class_id),
  assignments(class_id, teacher_id), submissions(assignment_id, student_id),
  chat_sessions(student_id), chat_messages(session_id)

  9. AI FEATURES
================================================================================

  AI MODEL
  ─────────────────────────────────────────────────────────
  Model:     openai/gpt-4o
  Provider:  GitHub Models inference endpoint
             (https://models.github.ai/inference)
  Auth:      GITHUB_TOKEN environment variable
  SDK:       openai npm package (configured with custom baseURL)

  FEATURE 1: AI MENTOR CHAT (streaming)
  ─────────────────────────────────────────────────────────
  Two modes selectable in the chat UI:

  Study Mode (default)
    System prompt: "Sahayak" — academic mentor persona
    Capabilities:
      – Explains concepts in structured, concise form
      – Renders LaTeX math inline ($...$) and display ($$...$$)
      – Renders markdown: tables, code blocks, bullet points
      – Aware of uploaded PDF content (appended to system prompt)
      – Aware of class study materials when launched from a class page
      – Conversation history: last 20 messages sent for context
    Streaming: SSE (Server-Sent Events) — tokens streamed token-by-token

  Support/Wellbeing Mode
    System prompt: empathetic counsellor persona
    Capabilities:
      – Active listening, emotional validation
      – Academic stress / anxiety management guidance
      – Encourages healthy habits
      – Redirects to professional help when appropriate
      – Safety flag: prepends [CONCERN_FLAG] if self-harm/crisis detected
    Academic questions are gently redirected back to Study Mode

  FEATURE 2: AI ASSIGNMENT GENERATION
  ─────────────────────────────────────────────────────────
  Teachers provide:
    – Topic name
    – Optional study material context (pasted text, up to 6,000 chars)
    – Assignment type: MCQ | Written | Mixed
    – Number of questions (e.g. 5, 10)
    – Difficulty: Easy | Medium | Hard

  GPT-4o returns structured JSON:
    { questions: [
        { question_text, type, options (MCQ only), correct_answer, points }
      ]
    }
  Questions are pre-populated into the assignment form for editing.

  FEATURE 3: AI WRITTEN ANSWER EVALUATION
  ─────────────────────────────────────────────────────────
  Teachers trigger AI evaluation on a submitted assignment.
  For each written answer, GPT-4o receives:
    – The question text
    – The model/expected answer
    – The student's actual answer
    – Maximum points available

  GPT-4o returns:
    { results: [{ answerId, score: 0–maxPoints, feedback: "..." }] }

  Results are saved to answers.score and answers.ai_feedback.
  Teachers can override scores manually.

  FEATURE 4: PDF DOCUMENT CHAT
  ─────────────────────────────────────────────────────────
  Students can upload a PDF in the chat interface.
  The backend extracts text using pdf-parse / pdfjs-dist (up to 100,000 chars).
  Extracted text is appended to the AI system prompt for the session.
  The AI can then answer questions specifically about the document.

================================================================================
  10. AUTHENTICATION & SECURITY
================================================================================

  • Authentication: Supabase Auth (email + password)
  • Sessions: JWT tokens managed by Supabase SDK, stored in localStorage
  • Password minimum: 6 characters (enforced on frontend)
  • Backend auth middleware: verifies JWT from Authorization header using
    Supabase service role to authenticate requests
  • Role-based access: requireRole() middleware checks user's role
    before allowing access to protected routes
  • Rate limiting: 100 requests per 15 minutes per IP (express-rate-limit)
  • CORS: configurable FRONTEND_URL env variable (whitelist)
  • Request body limit: 2MB (express.json limit)
  • Multi-tenancy isolation: All DB queries include institution_id filtering
    via RLS — users physically cannot read another institution's data
  • Registration flow: backend creates Supabase auth user first, then
    inserts into public.users with institution lookup/creation

  14. RUNNING THE PROJECT

  PREREQUISITES
  ─────────────────────────────────────────────────────────
  • Node.js 20+

  • A GitHub Personal Access Token with access to GitHub Models

  SETUP
  ─────────────────────────────────────────────────────────
  1. Clone the repository
     git clone <repo-url>
     cd ai-mentor

  2. Set up the database
     Open Supabase SQL Editor → paste contents of supabase/schema.sql → Run

  3. Configure environment variables
     cp backend/.env.example backend/.env    (fill in your values)
     cp frontend/.env.example frontend/.env  (fill in your values)

  RUN LOCALLY (without Docker)
  ─────────────────────────────────────────────────────────
  # Terminal 1 — Backend
  cd backend
  npm install
  npm run dev          # starts on http://localhost:3000 with hot-reload

  # Terminal 2 — Frontend
  cd frontend
  npm install
  npm run dev          # starts on http://localhost:5173

  RUN WITH DOCKER COMPOSE
  ─────────────────────────────────────────────────────────
  docker-compose up --build
  # Backend:  http://localhost:3000
  # Frontend: http://localhost:5173

  BUILD FOR PRODUCTION
  ─────────────────────────────────────────────────────────
  cd frontend
  npm run build        # outputs to frontend/dist/
  npm run preview      # local preview of production build

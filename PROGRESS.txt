PROJECT OVERVIEW
----------------
AI-Mentor is a full-featured, AI-powered Learning Management System (LMS) for
educational institutions. It supports three roles — Admin, Teacher, and Student
— and is powered by GPT-4o via the GitHub Models API.

Tech Stack:
  Frontend  : React 18 + TypeScript + Vite + Tailwind CSS + React Router v7
  Backend   : Node.js + Express 5 + OpenAI SDK
  Database  : Supabase (PostgreSQL + Auth + Storage)
  AI Model  : openai/gpt-4o via https://models.github.ai/inference
  State Mgmt: Zustand + React Query (client), Supabase RLS (server)

1.1 DATABASE SCHEMA (supabase/schema.sql)
-----------------------------------------
Created 12 PostgreSQL tables with Row-Level Security (RLS):
  - institutions    : id, name, logo, code (6-char unique), created_at
  - users           : id, institution_id, email, role, name, avatar_url, created_at
  - classes         : id, institution_id, name, subject, teacher_id, created_at
  - enrollments     : id, class_id, student_id, enrolled_at
  - study_materials : id, class_id, teacher_id, title, type, file_url, external_url
  - assignments     : id, class_id, teacher_id, title, type, status, deadline, total_points
  - assignment_targets: id, assignment_id, target_type, target_id
  - questions       : id, assignment_id, question_text, type, options(JSON), correct_answer, points, order_index
  - submissions     : id, assignment_id, student_id, submitted_at, total_score, status
  - answers         : id, submission_id, question_id, student_answer, score, ai_feedback, teacher_feedback
  - chat_sessions   : id, student_id, class_id, title, created_at
  - chat_messages   : id, session_id, role, content, timestamp

RLS helper functions:
  - my_institution_id() — returns the calling user's institution
  - my_role()           — returns the calling user's role

Performance indexes on: institution_id, class_id, student_id, assignment_id

3.1 STREAMING CHAT RESPONSES (Token-by-Token SSE)
-------------------------------------------------
Files changed:
  backend/src/services/openaiService.js  — added generateChatResponseStream()
  backend/src/controllers/chatController.js — added handleStreamChat (SSE endpoint)
  backend/src/routes/apiRoutes.js       — added POST /api/chat/stream
  frontend/src/hooks/useChat.js          — rewritten to consume SSE stream

How it works:
  1. Frontend sends POST /api/chat/stream with {message, history, pdfText, classId, mode}
  2. Backend sets Content-Type: text/event-stream and calls OpenAI with stream:true
  3. Each token chunk is written as: data: {"content":"..."}\n\n
  4. On completion: data: {"done":true}\n\n
  5. Frontend uses fetch() + ReadableStream + TextDecoder to parse SSE chunks
  6. Tokens are appended progressively to the AI placeholder message
  7. AbortController cancels in-flight requests on new messages

3.2 AI MENTOR WITH CLASS MATERIAL CONTEXT
------------------------------------------
Files changed:
  backend/src/controllers/chatController.js — fetches study_materials for classId
  backend/src/services/openaiService.js     — injects classContext into system prompt
  frontend/src/App.jsx                       — reads ?class= URL param
  frontend/src/pages/student/ClassDetail.tsx — "Chat with AI" button → /student/chat?class=<id>

How it works:
  1. Student clicks "Chat with AI" from a class page
  2. URL becomes /student/chat?class=<classId>
  3. Chat hook receives classId, passes it to streaming endpoint
  4. Backend fetches up to 10 study material titles/types for that class
  5. Context injected into system prompt:
     "Class study materials available: - Introduction to Algebra (pdf) - ..."
  6. GPT-4o uses this to give context-aware answers

3.3 AI ASSIGNMENT GENERATOR
----------------------------
New files:
  backend/src/controllers/aiController.js — generateAssignment()
  backend/src/routes/aiRoutes.js          — POST /api/ai/generate-assignment (teacher/admin)

Frontend changes:
  frontend/src/pages/teacher/AssignmentForm.tsx — collapsible "Generate with AI" panel

How it works:
  1. Teacher opens the "Generate with AI" panel (Sparkles icon)
  2. Enters topic/prompt, number of questions (1-15), difficulty (easy/medium/hard)
  3. POST /api/ai/generate-assignment with {classId, topic, type, numQuestions, difficulty}
  4. Backend calls GPT-4o to generate structured questions JSON
  5. Questions are appended to the form (existing non-empty questions are preserved)
  6. Teacher can edit, add, remove before saving/publishing

API payload format:
  POST /api/ai/generate-assignment
  Body: { classId, materialIds[], topic, type, numQuestions, difficulty }
  Response: { questions: [{ question_text, type, options, correct_answer, points }] }

3.4 AI WRITTEN ANSWER EVALUATOR
---------------------------------
New files:
  backend/src/controllers/aiController.js — evaluateSubmission()
  backend/src/routes/aiRoutes.js          — POST /api/ai/evaluate-submission/:submissionId

Frontend changes:
  frontend/src/pages/teacher/AssignmentDetail.tsx — "AI Grade" button per submission
    Shows ai_feedback under each written answer

How it works:
  1. Teacher sees "AI Grade" button on submissions with ungraded written answers
  2. POST /api/ai/evaluate-submission/:submissionId (no body needed)
  3. Backend fetches all null-score written answers for that submission
  4. Calls GPT-4o with {questionText, studentAnswer, modelAnswer, maxPoints}
  5. AI returns {answerId, score, feedback} for each answer
  6. Scores and ai_feedback saved to answers table
  7. total_score recalculated and saved to submission
  8. Frontend updates UI with scores + indigo feedback bubbles

3.5 AI PSYCHOLOGICAL SUPPORT MODE
-----------------------------------
Files changed:
  backend/src/config/constants.js — added SUPPORT_SYSTEM_PROMPT
  backend/src/services/openaiService.js — mode-aware system prompt selection
  backend/src/controllers/chatController.js — passes mode to streaming service
  frontend/src/App.jsx — Study Mode / Support Mode toggle buttons

Features:
  - Toggle between "Study Mode" (academic) and "Support Mode" (wellbeing)
  - Support mode uses a warm, empathetic system prompt
  - [CONCERN_FLAG] marker in AI response for serious distress signals
  - Support mode does NOT answer academic questions (redirects to Study Mode)
  - Mode indicator shown in top bar
  - Mode is passed with every message to the streaming endpoint

SUPPORT_SYSTEM_PROMPT highlights:
  - Warm, non-judgmental listener persona
  - Encourages healthy habits (sleep, exercise, social connection)
  - Suggests professional help when appropriate
  - [CONCERN_FLAG] prefix for crisis situations (self-harm, abuse)
  - Redirects academic questions to Study Mode

SETUP INSTRUCTIONS (Required before running)
================================================================================

1. Create a Supabase project at https://supabase.com

2. Run supabase/schema.sql in the Supabase SQL Editor
   (Project Settings → SQL Editor → New Query → paste schema.sql → Run)

3. Create a Storage bucket:
   - Name: study-materials
   - Set to Public (for student access to uploaded files)

4. Copy environment files and fill in your values:
   cp frontend/.env.example frontend/.env
   cp backend/.env.example backend/.env

   frontend/.env:
     VITE_SUPABASE_URL=https://<your-project>.supabase.co
     VITE_SUPABASE_ANON_KEY=<your-anon-key>
     VITE_API_URL=http://localhost:3000

   backend/.env:
     SUPABASE_URL=https://<your-project>.supabase.co
     SUPABASE_SERVICE_ROLE_KEY=<your-service-role-key>
     GITHUB_TOKEN=<your-github-models-token>
     PORT=3000

5. Install dependencies:
   cd frontend && npm install
   cd backend && npm install

6. Run the app:
   Terminal 1: cd backend && node server.js
   Terminal 2: cd frontend && npm run dev
